Introduction to ggplot2

Posted by Hamza Zubair in Hitchhikers Guide to ggplot2   

This post is a part of a series of posts on Chapters from the book The Hitchhiker's guide to ggplot2 in R. The book is available for free on Leanpub.com.¶

ggplot2 is an R package developed by Hadley Wickham. The package is based on an earlier work done by Leland Wilkinson who is a professor of Computer Science. Wilkinson developed something known as the Grammar of Graphics, which is an extensive set of rules and definitions for data visualizaiton. Wilkinson's grammar was further developed, enhanced and implemented in this package by Wickham. A must read paper by Wickham on the implementation of Grammar of Graphics is available online: A Layered Grammar of Graphics. To understand the intricacies of the package and to master it's nuances I would recommend reading his book: ggplot2: Elegant Graphics Data Analysis.

To understand anything about the ggplot2 package and how it works and to master it for visualization it is essential that we understand the grammar of graphics used in it. ggplot2 package is vast with a lot of functions and a lot of power, but we'll see how understanding the underlying grammar makes everything simple.

According to the grammar, every plot in ggplot2 has three key components:

  1. data
  2. aes
  3. geom

data you know, geom (geometries) is the visual elements of a plot, things that we see on a plot, like points, lines, ribbons, area, text, etc; aes (aesthetic mapping) is how the data connects to or maps on to the geoms, like axes, shapes, colors, sizes, etc. So, basically data is what you don't see, and geom is what you do see, and aes is how we connect the two. I cannot make it simpler than that.

Now, we will see how to use these three: data, aes and geom to create beautiful and meaningful plots. Let's load the package first. If you don't have ggplot2 installed you have to install.packages('ggplot2')

In [4]:
library(ggplot2)
In [5]:
ggplot()

This is what ggplot() does. Just creates an empty space where we can draw our plot. Now let's add some data. The data we will use is the mtcars dataset that comes pre-loaded with R.

So now let's put data in our ggplot() function

In [9]:
ggplot(data=d)

Does nothing. because as I said three key elements to building a plot. Data, aes and geom. So let's make some aesthetic mappings. As I said earlier, we can map our data onto axes, color, size, shape, etc. These are the elements of aesthetic mappings. So let's map the wt and mileage to x and y axes.

In [8]:
ggplot(data=d,aes(x=wt,y=mpg))

At this point all you see is the scales and axes. Because the third component, the thing that you actually see is missing. So let's add the third component. Now, with the data and the aesthetic mappings we sort of have the first layer of the plot. So the visual elements is a layer that needs to go on top of the base layer. We do that by adding a $+$ symbol.

In [13]:
ggplot(d,aes(wt,mpg)) + geom_point()

Notice, how we shortened the code by ignoring the argumens data, x and y. The first argument of ggplot() function always is the data, and the first two arguments of aes() are x and y variables. Another feature of ggplot is that you can store each layer in a variable like this:

In [17]:
l1 = ggplot(d,aes(wt,mpg))
In [15]:
l1 + geom_point()
In [16]:
l1 + geom_line()

Obviously the line plot doesn't make much sense in this situation. Knowing what to use when is your job. The package can't help with that.
