Introduction to ggplot2

in Hitchhikers Guide to ggplot2   

This post is a part of a series of posts on Chapters from the book The Hitchhiker's guide to ggplot2 in R. The book is available for free on Leanpub.com.¶

ggplot2 is an R package developed by Hadley Wickham. The package is based on an earlier work by Leland Wilkinson who is a professor of Computer Science at University of Illinois. Wilkinson developed something known as the Grammar of Graphics, which is an extensive set of rules and definitions for data visualizaiton. Wilkinson's grammar was further developed, enhanced and implemented in this package by Wickham. A must read paper by Wickham on the implementation of Grammar of Graphics is available online: A Layered Grammar of Graphics. To understand the intricacies of the package and to master it's nuances I would recommend reading his book: ggplot2: Elegant Graphics Data Analysis.

To understand anything about the ggplot2 package and how it works and to master it for visualization it is essential that we understand the grammar of graphics used in it. ggplot2 package is vast with a lot of functions and a lot of power, but we'll see how understanding the underlying grammar makes everything simple.

According to the grammar, every plot in ggplot2 has the following components:

  1. Layers
    1. Data
    2. Aesthetic Mapping
    3. Geometries
    4. Stat
    5. Position
  2. Scales
  3. Coordinates
  4. Facets
  5. Themes

Of the primary five components: Layers, Scales, Coordinates, Facets and Theme, Layers is the most important. Theme is just the beautification part. Facets is an advanced function, when you want to create not one, but several plots at once, using the same data dividing it into categories. We'll see how it's done in the future. Coordinates, for the most part need not be changed from Cartesian which is the default, so we can leave that discussion for some other time. Scaling is pretty much automatically handled, which rarely requires fiddling with, it's examples are continuous scaling or descrete scaling, etc. Topic for the future. So let's talk about the most important part of the plot, Layers.

Layers as you can see is again made up of five components: Data, Aesthetic Mapping, Geometries, Stat and Position. Of these stat and position are somewhat advanced, which are by default set to "identity", we'll discuss about them later. So finally and eventually we are left with the most basic and the most important components of a plot: Data, Aes (short for Aesthetic mapping), and Geom (short for Geometries). data you know what it is, preferably in the form of table, dataframe or just variables with values in them. geoms (geometries) are the visual elements of a plot, things that we see on a plot, like points, lines, curves, boxes, etc; aes (aesthetic mapping) are components that connect data with geoms. Examples of aes are the x-axis, the y-axis, shape, color, size, etc. So, basically data is what you don't see on a plot, and geom is what you do see on a plot, and aes is how we connect the two. I cannot make it simpler than that.

Now, we will see how to use data, aes and geom to create beautiful and meaningful plots. Let's load the package first...

In [7]:
library(ggplot2)

If you don't have ggplot2 installed, you have to install.packages('ggplot2').

The most basic command of the plot is...

In [5]:
ggplot()

And this is what it does... nothing. To see a plot we need to add a layer on top of this space. Remember layer? the most important component of a plot. A layer however requires these five components: data, aes, geom, stat, position. So let's load our data.

To see the datasets you have in the packages that are loaded in your current session: data(). However if you want to see all the data sets that are available in your system, whethere currently loaded or not, you can: data(package = .packages(all.available = TRUE))

From 8602 to 187, that's better. So, ggplot() produced nothing, which is why we need a layer.

This is what a layer funciton looks like...

In [140]:
layer(data=dal,mapping=aes(x=date,y=sales,color=month),geom="point",stat="identity",position="identity")

mapping: x = date, y = sales, colour = month 
geom_point: na.rm = FALSE
stat_identity: na.rm = FALSE
position_identity

Nothing happened, no plot was created, it just spat out the inputs as it is. Because to see a plot we need to add the layer on top of our base command ggplot(). We can do that using the $+$ symbol. So let's do that...

In [139]:
ggplot() + layer(data=dal,mapping=aes(x=date,y=sales,color=month),geom="point",stat="identity",position="identity")

You could also for example use "line" instead of a "point" as your geom...

In [142]:
ggplot() + layer(data=dal,mapping=aes(x=date,y=sales,color=month),geom="line",stat="identity",position="identity")

That was lot of code for a very simple plot, isn't it? ggplot2 has a solution to that and instead of using the layer function, we can use a shortcut the geom_*() function. There is a geom function for every geometry, so for the first plot you could...

In [143]:
ggplot() + geom_point(data=dal,aes(x=date, y=sales, color=month))

And unless you want to change stat and position, you don't need to use it. They are set to "identity" by default. For the second plot you could...

In [144]:
ggplot() + geom_line(data=dal,aes(x=date,y=sales,color=month))

In ggplot you could also store the outputs into variables...

In [145]:
base = ggplot()
points = geom_point(data=dal,aes(x=date,y=sales, color=month))
lines = geom_line(data=dal,aes(x=date,y=sales,color=month))
In [146]:
base + points
In [150]:
base + points + lines

This is why Wickham called it - layered grammar. The "points" layer was the first layer, and the "lines" layer was the second layer drawn on top of the first layer. The way the grammar works, it makes it extremely flexible for you to change anything. For example you could change the data set, the x and y mappings or the colors of the second layer, thereby increasing the complexity or perhaps the utility of your plot. We will do that in the coming articles.

Shortcut¶

There is even a shorter way of doing what we just did. We did it not because it was the best way of creatign plots in ggplot but because that showed us how the ggplot's grammar works.

You noticed how we had to enter the data, x, y, color arguments twice, once for geom_point and then again for geom_line. If you are sure you are not going to change them in the subsequent layers you can infact add them in the base layer like this...

In [152]:
base = ggplot(dal,aes(date,sales, color=month))
In [153]:
base + geom_point()
In [154]:
base + geom_point() + geom_line()

This probably gives you a clue of the power of ggplot2. Empty geom_point() and empty geom_line() functions meant that it had it to take its arguments from the previous layer. And so both functions used the parameters supplied in the ggplot() function.
